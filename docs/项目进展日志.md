## 📁 项目A：AI竞品分析助手（Dify）

### 2026-04-02

**完成事项**

- **模块2 方案B工作流（Dify）已完成**
  - 在 Dify 中新建模块2工作流，使用 Brave Search API 实现自动化用户评论搜索
  - 节点结构：开始 → 代码执行（构造搜索词）→ HTTP请求（Brave API）→ LLM（聚类分析）→ 结束
  - 已优化为多轮搜索（负面+正面关键词），提高召回质量

- **用户评论采集API服务已部署上线**
  - 在腾讯云服务器上部署自建 Python 评论采集 API（Flask），端口5100
  - 采集渠道：Brave Search API（主力，每次约30-40条）+ Apple App Store RSS（辅助，每次约100条）
  - 已放弃小红书/微博（反爬严重，返回垃圾数据）
  - 功能特性：API Key 验证、中文编码修复、错误处理、渠道独立降级
  - 测试通过：微信（134条）、飞书（138条）均正常返回

- **模块2 评论分析工作流（Dify）进行中**
  - 已配置 HTTP 请求节点调用腾讯云评论采集 API

**待办**

- 完成模块2 LLM 聚类分析节点配置并测试
- 验证 SSL 证书设置（http 连接需关闭 SSL）
- 测试完整工作流：输入产品名 → 自动采集 → 聚类分析 → 输出痛点报告
- 考虑模块1和模块2串联测试
- 服务稳定性：确认重启后 review-api 自动启动（可加 systemd 服务）

**关键决策**

- Dify 作为面试展示主平台，OpenClaw 作为补充技术信用
- 放弃小红书/微博，聚焦 Brave Search + App Store 两个稳定渠道
- 坚持全自动化采集→分析闭环，拒绝知识库手动灌数据方案
- 迭代故事线：方案A（手动）→ 方案B（Brave搜索）→ 方案C（自建采集API）
